docker笔记

2021-10-27 00:28:11
Categories： Tags：

docker

本博客仅为个人笔记

简介

官方文档https://docs.docker.com/get-docker/

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包（环境）到一个可移植的镜像中，然后发布到任何流行的 Linux或Windows 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

镜像、容器和仓库

镜像（images）、容器（container）和仓库（DockerHub）

个人理解：镜像就是安装文件，容器安装后的运行进程、仓库互联网备份。

容器与虚拟机的区别 ：
一个容器中运行原生Linux和共享主机与其它容器的内核，它运行一个独立的进程，不占用任何其它可执行文件的内存，使其轻量化。

仓库

仓库（DockerHub）地址hub.docker.com，可以去获取自己想要的公开镜像

镜像命令

  • 镜像信息
docker images
docker images -a 展示所有镜像
docker images -q 只展示镜像的ID
REPOSITORY TAG IMAGE ID CREATED SIZE
镜像的仓库源 镜像的标签 镜像的ID 镜像的创建时间 镜像的大小
  • 搜索镜像 
docker search 镜像
docker search 镜像 --filter=starts=3000
  • 下载镜像 
docker pull 镜像 ：默认下载最新版本（latest）
docker pull 镜像:版本 ：下载指定的版本
//docker pull mysql:8.0.19
  • 删除镜像
docker rmi -f 镜像ID：删除镜像

容器

  • 启动 
docker run [可选参数] 镜像
可选参数：
--name="名称"        // 设置容器名
-d                     // 后台方式运行
-it                 // 使用交互方式运行，可以进入容器查看内容
-p                    //（小p）, 指定容器端口
    -p ip:主机端口:容器端口
    -p 主机端口:容器端口（常用）
    -p 容器端口
-P                    //（大P）, 随机指定容器端口

#将nginx的80端口映射到宿主机的800端口上
docker run -d -it -p 800:80 nginx 

docker exec -it e5 /bin/sh  //进容器
exit     //退出
  • 容器信息 
docker ps命令 #列出当前正在运行的容器
-a  # 列出当前正在运行的容器+带出历史运行过的容器
-n=? # 显示最近创建的容器
-q  # 只显示容器的编号
  • 启动和停止 
启动和停止容器的操作

docker start 容器id     #启动容器
docker restart 容器id  #重启容器
docker stop 容器id      #停止当前正在运行的容器
docker kill 容器id      #强制停止当前容器
  • 删除 
docker rm 容器id

容器其他操作


docker exec -it e5 /bin/sh  //进容器
exit     //退出

//帮助重新配置端口映射关系。
docker commit -m="描述信息" -a="作者" 容器id 目标镜像名:[TAG]
docker run --name container02 -p 80:80 new_image:tag
=============
docker logs -tf --tail number 容器ID
-tf： 显示日志
--tail number ：显示日志尾部的几条日志
=============
查看docker容器内部的进程信息
docker top 容器ID
